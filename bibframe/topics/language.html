Language of Part
BIBFRAME Topics and Issues

Back to Topics and Issues Home

A Work  might have different parts with different languages, for example a book written in English might have an abstract written in French.  BIBFRAME currently handles this by allowing the property bf:language to indicate what part of the Work the language applies to: property bf:languageOfPart is a property of bf:Language.  It’s value is a string, for example ‘abstract”.

It has been proposed that a part of a work that needs to be addressed in this manner be treated as a resource.

 The parts could be linked to/from the Work by properties hasPart and isPartOf. hasPart could have specific subproperties for the part types, like hasAbstract.

Example:

<http://bibframe.example.con/works/books/goldfinch> a bf:Work ;
      bf:title "The Goldfinch" ;
      bf:language  <http://id.loc.gov/vocabulary/iso639-1/en>
      bf:hasAbstract <http://bibframe.example.con/workParts/abstracts/goldfinch> .

<http://bibframe.example.con/workParts/abstracts/goldfinch> a bf:Abstract ;
      bf:ispartOf  < http://bibframe.example.con/works/books/goldfinch >.
      bf:language  <http://id.loc.gov/vocabulary/iso639-1/fr> .

In the above example, what sort of resource is "abstract"? It can certainly be represented as a separate resource, but is it a separate Work? Probably some of these part can be realized as separate Works, but most likely not all. In fact, some of these parts reeally cannot be realized as separate works.

Following is a very imcomplete list of parts, for consideration:

  • Summary or abstract. Separate Resource.
  • Sung or spoken text. Not a separate resource, another language in the work itself.
  • Librettos. Separate Resource.
  • Table of contents. Separate Resource.
  • Accompanying material other than librettos.. Separate Resource.
  • Original. Separate Resource.
  • Subtitles or captions. Not separate resources, but additional languages for this resource.
  • Intermediate translations. Separate Resource.
  • Original accompanying materials other than librettos. Separate Resource.
  • Original libretto. Separate Resource.

Questions:

  1. What are the parts of a Work for which we want to represent the language?
  2. For parts that can be represented as separate resources, which among these can be represented as Works?
  3. For a part that can be represented as a separate resource, but not a Work, should a new class bf:Parts be defined, with these parts as subclasses?
  4. For a part that cannot even be represented as a separate resource do we really need to single these out, and if so, how?
