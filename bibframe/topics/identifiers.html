Identifiers
BIBFRAME Topics and Issues

Back to Topics and Issues Home

A BIBFRAME resource may have several identifiers.  An Instance may have for example an  ISBN, ISSN, ISMN, and LCCN.  A Work might have an ISAN, ISTC, and ISWC. We refer to these as "identifier schemes". For example, for the ISSN: 0961-8368,  the identifier scheme is 'issn' and the identifier value is '0961-8368'.

For each identifier represented in the resource's description there is a bf:Identifier, a resource which records the value of the identifier, ,information about the identifier such as qualifier, assigners and  status, along with the identifer value.

Issues

  1. Av identifier is of little use unless the identifier scheme is known. The identifier scheme usually cannot be determined from the identifier value alone. How does (or should) BIBFRAME indicate the scheme?
  2. For a given identifier scheme, there is often a cannonical way to represent an identifer in that scheme via a URI. For example, issn '0961-8368' can be represented as urn:ISSN:0961-8368. What is the relationship between the strings '0961-8368' and 'urn:ISSN:0961-8368'?

Representing the Identifier Scheme

Using the example issn 0961-8368, there are several ways to represent it.

Here, the scheme is represented by the property bf:identifierScheme, as the string "issn".

  <http://www.example.com/works> a bf:Work ;
       bf:title                          "Protein Science" ;         
      bf:identifier [
               a                                           bf:Identifier ;
                bf:identifierScheme                "issn" ;
                bf:identifierValue                    "0961-8368" . ] .

Here, similarly, the scheme is represented by the property bf:identifierScheme, but instead of a string, a resource (http://id.loc.gov/vocabulary/identifiers/issn) is supplied.

  <http://www.example.com/works> a bf:Work ;
       bf:title                          "Protein Science" ;         
      bf:identifier [
               a                                           bf:Identifier ;
                bf:identifierScheme                "<http://id.loc.gov/vocabulary/identifiers/issn>" ;
                bf:identifierValue                    "0961-8368" . ] .

bf:issn is a subproperty of bf:identifier (used in the above examples). So here, the scheme is indicated by the property, and so bf:identifierScheme is not necessary.

  <http://www.example.com/works> a bf:Work ;
       bf:title                          "Protein Science" ;         
      bf:issn [
               a                                           bf:Identifier ;
                bf:identifierValue                    "0961-8368" . ] .

There are roughly 35 or so of these subproperties of bf:identifier, corresponding to identifier schemes.

Note: among these properties are bf:urn and bf:uri. These are not proper identifier schemes and these properties will probably be removed.

For those schemes not represented by a property, bf:identifierScheme may be used to indicate the scheme.

URI Representation

For a given identifier scheme, there is often a cannonical way to represent an identifer in that scheme via a URI. The benefit of supplying the URI representation is that it provides a uniform/normalized form for better performance. (It is not intended that by supplying the URI form there is any guarantee that the URI will resolve.)

'urn:ISSN:0961-8368' is the URI representation of issn '0961-8368':

  <http://www.example.com/works> a bf:Work ;
       bf:title                          "Protein Science" ;         
      bf:identifier [
               a                                           bf:Identifier ;
                bf:identifierScheme                "issn" ;
                bf:identifierValue                   "0961-8368" ;
                bf:uriRepresentation       “urn:ISSN:0961-8368”  . ] .

Note: the property bf:uriRepresentation is not yet part of the BIBFRAME vocabulary. It is proposed for addition.

If the string '0961-8368' is an identifier whose scheme is 'issn', what is the string 'urn:ISSN:0961-8368'? Is it another identifier for the same resource, with a different scheme? No, it is not. Its scheme cannot be 'urn' because there may be many identifiers (of different schemes) that have urn representations. ( A resource can have only one identifier in any given scheme.) For the same reason, the scheme cannot be 'uri'.

So, 'urn:ISSN:0961-8368' is not a separate identifier, it is simply said to be "the URI representation of issn 0961-8368".
