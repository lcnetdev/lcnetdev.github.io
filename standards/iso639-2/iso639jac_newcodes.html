Other considerations:

  1. References to languages contained in group language names or unused forms of names: see MARC list: http://lcnetdev.github.io/marc/languages (AFNOR comment on DIS).
  2. The statement in 4.1.5 requiring that ancient languages which are not given individual language codes are assigned the code for the major language collective to which an ancient language belongs rather than the code for the modern language which evolved from the ancient language is problematic (Germany, DIS). If there are specific ancient languages that warrant separate codes based on the number of documents existing, these may be brought to the attention of the Registration Authority.
    There may be additional language code requests between now and February.

The following are conflicts between ISO CD639-1 and ISO 639-2:

Language Code in 639-1 Code in 639-2
Cornish kw cor
Herero oh her
Kuanyama/Kwanyama ok kua
Ndonga on ndo

Issues about consistency between the two standards will be brought out in a discussion paper for consideration by the ISO 639/JAC representatives (Discussion of working principles for ISO 639 maintenance).

One language in CD639-1 is represented by 2 languages in 639-2:
Ndebele (639-1); Ndebele, North and Ndebele, South in 639-2. It is not clear whether the Ndebele in 639-1 is intended to include both languages or one of them.
