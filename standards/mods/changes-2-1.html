MODS: Changes for Version 2.1

The following are major changes to be incorporated into MODS in version 2.1. This is being considered a minor change and does not increment the main version number, because it does not result in invalidating any existing MODS records. These are based on comments, defect reports, and suggestions for enhancements as a result of experimentation. Where appropriate the MARC mapping is provided.

  1. Versioning. Allow for a version attribute at the top of the document to indicate the current major and minor version of the document (in this case 2.1).
    Indicate the major version of the schema in the namespace and in the filename.
    e.g. http://www.loc.gov/mods/2; filename: mods2.xsd
    For minor changes (i.e. those that don't invalidate existing records, i.e. enhancements or minor corrections): change the version attribute in the schema and increment the minor version e.g. 2.1 to 2.2. Include a change history under documentation in the schema. For major changes (new version): Document the change in change history; change target namespace (i.e. http://www.loc.gov/mods/3; new filename (mods3.xsd)

  2. Elements. Change the MODS schema to define all of the mods elements as global elements so that they can be imported into other schemas.

  3. name. Add type "termsOfAddress" to namePart (for titles, Jr., etc.). (X00$c)

  4. identifier: Add hdl to suggested value list in documentation under <identifier>, to be treated like "doi". (Note that if either become an approved URI scheme or URN namespace, the method of representating them may change.)

  5. originInfo.
    • Allow for multiple <place> element and subelements <code> and <text> to be repeatable, in any order. (This was an error in the previous schema.)
    • Add <dateValid>
    • Add <dateModified>

  6. recordInfo. <recordContentSource> needs an authority attribute.

  7. relatedItem
    • Add displayLabel attribute
    • Add <part> as a subelement under relatedItem (as below)

partDescription: Allows for full citation information about the location within a host item; use with relatedItem type="host".
