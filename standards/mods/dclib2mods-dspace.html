Element Qualifier Scope Note MODS mapping
contributor   A person, organization, or service responsible for the content of the resource. Catch-all for unspecified contributors.

<name><namePart>

If you designate type of name (personal, corporate) add type="personal" etc. under <name>
contributor advisor Use primarily for thesis advisor. <name><role><roleTerm type="text">advisor
contributor author   <name><role><roleTerm type="text">author
contributor editor   <name><role><roleTerm type="text">editor
contributor illustrator   <name><role><roleTerm type="text">illustrator
contributor other  

<name><role><roleTerm type="text">other

Alternatively, could leave out role.
coverage spatial Spatial characteristics of content. <subject><geographic>
coverage temporal Temporal characteristics of content. <subject><temporal>
creator   Do not use; only for harvested metadata.  
date   Use qualified form if possible.

<originInfo><dateOther>

If using structured form, give encoding="iso8601" (YYYYMMDD) or "w3cdtf" (YYYY-MM-DD)
date accessioned Date DSpace takes possession of item. Not in MODS; we can add or you can use extension element (see below)
date available Date or date range item became available to the public. Not in MODS; we can add or you can use extension element (see below)
date copyright Date of copyright. <originInfo><copyrightDate>
date created Date of creation or manufacture of intellectual content if different from date.issued. <originInfo><dateCreated>
date issued Date of publication or distribution. <originInfo><dateIssued>
date submitted Recommend for theses/dissertations. Not in MODS; we can add or you can use extension element (see below)
identifier   Catch-all for unambiguous identifiers not defined by qualified form; use identifier.other for a known identifier common to a local collection instead of unqualified form. <identifier>
identifier citation Bibliographic citation for works that have been published as a part of a larger work, e.g. journal articles, book chapters.

<relatedItem type=”host”><part><text>

If elements of citation are parsed, can use additional MODS elements under <part>
identifier govdoc Government document number <identifier type="govdoc">

identifier issn International Standard Serial Number <identifier type="issn">
identifier sici Serial Item and Contribution Identifier <identifier type="sici">
identifier ismn International Standard Music Number <identifier type="ismn">
identifier other A known identifier type common to a local collection. <identifier type="local">
identifier uri Uniform Resource Identifier <identifier type=”uri”> (if persistent)
< location><url> (if http:)
repeat if both

Probably can assume it is a location url
description   Catch-all for any description not defined by qualifiers. <abstract>

Could use <note> if more appropriate
description abstract Abstract or summary. <abstract>
description provenance The history of custody of the item since its creation, including any changes successive custodians made to it.

<note type="provenance">

Note types are uncontrolled; we can add your note types to list at:

http://www.loc.gov/standards/mods/mods-notes.html
description sponsorship Information about sponsoring agencies, individuals, or contractual arrangements for the item. <note type="sponsorship">
description statementofresponsibility To preserve statement of responsibility from MARC records. <note type="statement of responsibility>

Note: this is in the MARC/MODS mapping this way!
description tableofcontents A table of contents for this item. <tableOfContents>
description uri Uniform Resource Identifier pointing to description of this item. <note xlink:simpleLink="content">

Place URI as value of attribute and close tag
format   Catch-all for any format information not defined by qualifiers. <physicalDescription><form>
format extent Size or duration. <physicalDescription><extent>
format medium Physical medium. <physicalDescription><form>

If value is from a controlled list, can use authority=""
format mimetype Registered MIME type identifiers. <physicalDescription><internetMediaType>
language   Catch-all for non-ISO forms of the language of the item, accommodating harvested values. <language><languageTerm>
language iso Current ISO standard for language of intellectual content, including country codes (e.g. "en_US"). <language><languageTerm authority="rfc3066"
If using 2-character language codes (there are 2 ISO standards, ISO 639-1 and 639-2; rfc3066 is a profile of ISO 639-1 which allows for additional 3-character codes from 639-2 where a 2-character code does not exist.
publisher   Entity responsible for publication, distribution, or imprint. <originInfo><publisher>
relation   Catch-all for references to other related items. <relatedItem>

Need to include an additional tag. If data with no qualifiers tends to be text, use <relatedItem><note>. If tends to be title, use <title>.
relation isformatof References additional phsical form. <relatedItem type=”otherFormat”>
relation ispartof References physically or logically containing item. <relatedItem type=”host”>
relation ispartofseries Series name and number within that series, if available. <relatedItem type=”series”>
relation haspart References physically or logically contained item. <relatedItem type=”constituent”>
relation isversionof References earlier version.

<relatedItem type=”preceding”>

Not clear what the difference is between this and replaces/isReplacedBy. MODS uses preceding and succeeding to show earlier version and otherVersion to show a related work differing in intellectual content (without regard to a point in time).

 
relation hasversion References later version. <relatedItem type=”succeeding”>

Note that hasVersion is not in DC-Lib
relation isbasedon References source. <relatedItem type=”original”>
relation isreferencedby Pointed to by referenced resource. <relatedItem type=”isReferencedBy”>
relation requires Reference resource is required to support function, delivery, or coherence of item. <note type=”requires”>
relation replaces References preceeding item. <relatedItem type=”preceding”>
relation isreplacedby References succeeding item. <relatedItem type=”succeeding”>
relation uri References Uniform Resource Identifier for related item. <relatedItem><location><url>
rights   Terms governing use and reproduction. <accessCondition type="useAndReproduction">
rights uri References terms governing use and reproduction. <accessCondition xlink:simpleLink="content">

Place URI as value of attribute and close tag
source   Do not use; only for harvested metadata. Note: this would be <relatedItem type="original">
source uri Do not use; only for harvested metadata. Note: this would be <relatedItem type="original" xlink:simpleLink="content">
subject   Uncontrolled index term. <subject><topic>
subject classification Catch-all for value from local classification system; global classification systems will receive specific qualifier. <classification>
subject ddc Dewey Decimal Classification Number <classification authority="ddc">
subject lcc Library of Congress Classification Number <classification authority="lcc">
subject lcsh Library of Congress Subject Heading <subject authority="lcsh">

If the mapping from MARC to MODS can be followed, use subtags as specified. If not parsing parts of LCSH, use <subject><topic> with LCSH string
subject mesh Medical Subject Headings <subject authority-"mesh">
subject other Local controlled vocabulary. <subject authority-"local">
title   Title statement/title proper. <titleInfo><title>
title alternative Varying (or substitute) form of title proper appearing in item, e.g. abbreviation or translation. <titleInfo type=”alternative”><title>
type   Nature or genre of content. <typeOfResource> (if using high level types from controlled MODS list.)
<genre> if a specific genre type
