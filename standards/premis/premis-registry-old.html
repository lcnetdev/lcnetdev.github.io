DigiTool
Ex Libris

Description:

DigiTool is an enterprise solution for the management of digital assets in libraries and academic environments.

DigiTool consists of a number of modules, each designed to address different needs, functions, and workflows pertaining to the life cycle of a digital object, from depositor ingest into the DigiTool Repository, through administration, metadata editing, and collection management, to end-user search and the delivery of objects.

In the system there is a built in support for the history event entity and the object entity from the PREMIS schema, and we are working to add the rights entity in the near future - the customer can choose whether he would like to use the mentioned schema or not.

DigiTool is currently in use at over 60 institutions worldwide, as an institutional repository and for preservation purposes.

Web site:

n/a

Dates:

The first implementation was May 2002.

Documentation:

http://www.exlibrisgroup.com/digitool_presentations.htm

Tools used:

JHOVE - to extract technical metadata
URN-NBN - to support the German persistent identifier system.
METS - to describe complex objects and to be used as a delivery method

National Snow and Ice Data Center Repository
National Snow and Ice Data Center

Description:

The OAIS reference model is one of NSIDC's guiding documents as an archive. Several years ago, we decided to use the OAIS reference model in developing a schema for storing preservation information about our science data sets - we already are NASA DIF and FGDC-compliant in that regard. In the meantime, the PREMIS group came out with their schema. As an experiment, since implementation was still ongoing, we chose to make our schema PREMIS compliant.

Web site:

None, this is an internal project

Dates:

Completion and transfer to operations is expected in April, 2006

Documentation:

We have an internal website but at this point it is not generally accessible. The publically accessible NSIDC data catalog can be found at http://nsidc.org/data/search/search.html

Tools used:

We have developed schema extensions to our central data catalog, along with web-based tools for population.

Contacts:

R. Duerr
rduerr@nsidc.org

Paradigm
Oxford University Library Services
John Rylands Library, University of Manchester

Description:

Focussing on the papers of contemporary politicians, the Paradigm project is exploring the issues surrounding the preservation of born-digital personal papers to ensure that they continue to be accessible to the researchers of the future. The project is testing various tools, including Fedora and DSpace, and will develop exemplar strategies and best-practice guidelines that will be of use to any institution which collects, preserves and maintains access to private papers.

Web site:

http://www.paradigm.ac.uk

Dates:

The project runs until February 2007. We plan to develop metadata profiles for digital object types that are commonly found in personal papers. We are exploring a number of schemes including FoXML, METS and PREMIS.

Documentation:

  

Tools used:

Testing various tools such as Jhove and the National Library of New Zealand Metadata Extraction Tool. Also using Fedora, which wraps supplied XML metadata and Fedora-generated metadata in its native FoXML but can export this metadata as METS files.

Contacts:

Susan Thomas
susan.thomas@bodley.ox.ac.uk

Stanford Digital Repository
Stanford University Libraries

Description:

The Stanford Digital Repository (or SDR) is a set of services of the Libraries which provides stewardship over information in digital form, maintaining access to that information over time. The SDR is designed to support the wide variety of digital information being created and used by Stanford communities engaged in learning, scholarship, and research. The PREMIS data dictionary and schemas are being used as part of the strategy to record preservation metadata in the SDR.
