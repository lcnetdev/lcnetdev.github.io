Changes and corrections to PREMIS data dictionary and schemas

Because the PREMIS data dictionary and schemas will remain stable until after the trial use period, this document records corrections (errata) and changes that have been proposed for the next revision.

The PREMIS Editorial Committee is actively working on revisions and will be documenting decisions about changes on this list.

Proposed changes (Data dictionary)

  • add clarification about repeatability of identifiers: objectIdentifier and agentIdentifier are repeatable because they may be defined and known outside the repository, and thus may have multiple identifiers; eventIdentifier and permissionStatementIdentifier are defined only inside the repository and should not be repeatable. (Sept. 16, 2005)
  • add an additional eventType suggested value for "digitization" (Sept. 16, 2005)
  • significantProperties needs to be defined outside of objectCharacteristics, since it is applicable at the representation level; objectCharacteristics applicable only for file and bitstream (discuss whether to also include under objectCharacteristics (Mar. 10, 2006)
  • make storage not applicable to bitstreams and define a new element for recording the location of a bitstream (Mar. 10, 2006)
  • define an unstructured keyInfo under signatureInformation to allow users to overlay established key structures (XML signatures) (see message in PIG list) (Mar. 10, 2006)
  • introduce a numbering system to be used for navigation for each semantic unit in the data dictionary. The numbering system will be grouped by entity types: Object semantic units will be 1. and Event Semantic Units will be 2. If Business rules are added, they will be 5. The numbers can be added as comments within the Schema..

 

Proposed changes (schemas)

  • significantProperties at higher level than objectCharacteristics (Mar. 10, 2006)
  • specify that creatingApplication, environment, and format cannot be empty but must include at least one element (or more than one) (Mar. 10, 2006)
  • define a new element for recording the location of a bitstream (Mar. 10, 2006)
  • allow for more control and extensibility in specifying object categories by following the schema suggestions in this message to the PIG list (Mar. 10, 2006)
  • define an unstructured keyInfo under signatureInformation as above (Mar. 10, 2006)

 

Errata (Data dictionary)

  • Change swVersion to not repeatable at file level (Mar. 10, 2006)
  • Change storage (container) to optional for bitstreams (Mar. 10, 2006)
