MODS: Tentative Changes for Next Version

The current version of MODS, 1.2, is undergoing a period of trial use until approximately the end of 2002. Version 1.2 will remain stable during that period while we invite comments, defect reports, and suggestions for enhancements as a result of experimentation.

We are maintaining a list, below, of tentative changes to be applied to the next version of MODS.

  1. PublicationTypeInfo includes elements dateIssued, dateCreated, and dateCaptured. otherDate will be added, as a generic date element for dates that don't fall into one of these categories.

  2. An attribute, keyDate, will be added to dateType so that a particular date may be distinguished among several dates. Thus for example when sorting MODS records by date, a date with this attribute would be the date to sort on. It would assume a value of "yes", and it should only occur for one date at most in a given record.

  3. In the genreType definition, the list of values (effectively a comment, with a <documentation> element) will be removed. It will be maintained outside the schema.

  4. Add additional values to <physicalDescription><form> (from 008/23): print; large print; regular print reproduction.

  5. Add <publicationInfo> to <relatedItem> as subelement.

  6. In <identifier>:
    Add attribute displayLabel (856$3)
    Add "stock number" as a value for type attribute

  7. Add attribute displayLabel in <location> (852$3)

  8. Add attributes lang (perhaps lang and xml:lang) and script to be used for any element

  9. Add <languageOfCataloging> under <recordInfo>
