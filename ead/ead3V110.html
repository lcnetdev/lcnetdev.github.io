Release of EAD3 v1.1.0

EAD users,

On behalf of SAA’s Technical Subcommittee for Encoded Archival Standards (TS-EAS) I am pleased to announce the release of EAD3 v1.1.0. EAD3 Version 1.1.0 represents the first minor revision to EAD3 since its initial release in 2015.

Schemas
EAD3 v1.1.0 is available for download from Github at: https://github.com/SAA-SDT/EAD3/releases/tag/v1.1.0. The release package can be downloaded as a .zip or .tar.gz, and contains the full set of EAD3 v1.1.0 schemas: Relax NG, W3C, and DTD; the ancillary Schematron schema for validating external code lists, standard data patterns, element or attribute co-occurrence; release notes; a readme file; and the license (CC0).

Tag Libraries
The official Library of Congress EAD site (http://www.loc.gov/ead/) has been updated with links to the EAD3 v1.1.0 schemas, the download package, and the revised EAD3 Tag Library (available in both HTML and PDF formats).

What’s new in EAD3 v1.1.0?
EAD3 v1.1.0 includes seven changes--one new element (rightsdeclaration), two schema bugfixes, and four datatype/attribute extensions. Note that changes introduced in EAD3 v1.1.0 will not invalidate existing EAD3 files.

For more details about the changes in EAD3 v1.1.0 see the release notes in Github, which also include links to a discussion thread for each change: https://github.com/SAA-SDT/EAD3/blob/master/RELEASE.md#new-in-ead3-110

Changes in EAD3 v1.1.0 reflect feedback from the community since the initial EAD3 1.0.0 release in 2015. The EAD3 v1.1.0 release also represents a move towards a more agile schema maintenance plan in which smaller updates and fixes can be released on a more regular basis. In the next few months, TS-EAS will be working on documentation outlining this new incremental release process.

Finally, special thanks to members of the TS-EAS Schema subteam, Documentation subteam, and everyone else who participated or commented on EAD3 v1.1.0.

Best,
Noah Huffman
TS-EAS, EAD subteam lead
