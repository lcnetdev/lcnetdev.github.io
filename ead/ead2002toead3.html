EAD2002 to EAD3 Beta Conversion Stylesheet Available

[Released 2016-01-15]

All,

I am happy to announce the Beta release of the EAD 2002 to EAD3 migration style sheet.

The beta release is available on GitHub at https://github.com/SAA-SDT/EAD2002toEAD3/releases/tag/v1.0.0-beta. 

Style sheet documentation is available in the README file, which is included in the release package and is accessible at https://github.com/SAA-SDT/EAD2002toEAD3/blob/master/README.md. 

The style sheet has been tested extensively against a wide variety of instances of EAD 2002. It is, however, a beta release. Known bugs are documented in the GitHub repository issue tracker: https://github.com/SAA-SDT/EAD2002toEAD3/issues. If you discover additional bugs or identify useful enhancements, please report them in the GitHub issue tracker.

The migration style sheet should work with most valid EAD 2002 instances. DTD-valid EAD 2002 instances containing entity references, however, may fail or result in validation errors. Also, EAD 2002 instances that are not valid against the DTD or schema, or which contain local encoding variations, will likely not result in valid EAD3 instances.

Finally, thank you for your patience as we worked through the many, complicated details involved in migrating from EAD 2002 to EAD3. If you have any questions regarding migration paths from EAD 2002 to EAD3, I will be happy to provide clarification.

Best,

Mike Rush

TS-EAD Co-chair
